HTML Layout Elements (New Semantic Elements in HTML5)

Semantic Elements?

Semantic elements are elements with a meaning and clearly describes its meaning to both the browser and the developer.

  • Semantic elements:

    <form>, <table>, <img> and etc. - Clearly defines its content.

  • Non-semantic elements:

    <div></div>, <span> and etc. - Tells nothing about its content.

HTML5-New semantic elements to define different parts of a web page:

  • <header> - Specifies a header for a document or section.

  • <nav> - Defines a set of navigation links.

  • <section> - Element defines a section in a document.

  • <article> - Element specifies independent/self-contained content.

  • <aside> - Content which is related to the surrounding content.

  • <footer> - Element specifies a footer for a document or section.

  • <figure> - Element to mark up a photo in a document.

  • <figcaption> - Specifies caption to an image, grouped in <figure>

  • <details> -Defines additional details that user can view or hide

  • <summary> -Defines a visible heading for a <details> element

  • <main> - Specifies the main content of a document

  • <mark> - Defines marked/highlighted text

  • <time> - Defines a human-readable date/time.

  • <hgroup> - Grouping titles with their associated subtitles
